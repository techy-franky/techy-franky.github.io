A03:2021 - Injection

Category Explanation: Injection flaws occur when untrusted data is sent to an interpreter as part of a command or query, allowing attackers to execute unintended commands or access unauthorized data. This is one of the most common and dangerous vulnerabilities, often stemming from poor input validation. Real-world examples include SQL injection (SQLi), NoSQL injection, command injection, and LDAP injection. In pentesting, we exploit these by crafting malicious inputs to manipulate backend queries, potentially leading to data leaks, unauthorized access, or even remote code execution (RCE). Mitigation involves using parameterized queries, escaping inputs, and web application firewalls (WAFs).

Lab Title: SQL Injection Vulnerability Allowing Login Bypass

Description: This lab simulates a login form vulnerable to SQL injection in the username field, allowing an attacker to bypass authentication without knowing valid credentials. The goal is to log in as the administrator user.

Steps to Exploit:

  1. i Navigate to the login page and intercept the POST request using Burp Suite.
  2. i Tested the username parameter for SQLi by injecting a single quote (') to trigger an error, confirming sqli vulnerability.
  3. i then Crafted a payload to make the SQL query always true: username=administrator'--&password=anything
  4. then i Forwarded the modified request to bypass login and access the admin panel.

Payload/Technique: administrator'-- (comments out the rest of the query, ignoring the password check).

Impact: In a real app, this could lead to unauthorized access to sensitive data, account takeovers, or full system compromise if combined with other vulns.

Remediation: Use parameterized queries or prepared statements in the backend code. Sanitize inputs and implement proper error handling to avoid leaking database info.
